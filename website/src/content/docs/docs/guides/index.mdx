Welcome to the Distr guides section. Here you'll find practical, step-by-step tutorials covering everything from onboarding applications to managing customer deployments.

## Getting Started

Start with these essential guides to set up your first applications and customers:

- **[Run Distr on macOS](/docs/guides/distr-on-macos/)** - Configure Docker Desktop for Distr on macOS
- **[Create Application](/docs/guides/application/)** - Create Docker and Helm applications, add versions, and manage application templates
- **[Create a Deployment](/docs/guides/deployment/)** - Deploy applications to customer environments using the updated deployment flow
- **[Onboarding a New Customer](/docs/guides/customer-management/)** - Create customer accounts and invite users; see [RBAC](/docs/product/rbac/#inviting-customer-organization-users) for roles and permissions
- **[Container Registry](/docs/guides/container-registry/)** - Configure your registry and start distributing software

## Configuration & Customization

Configure your applications and customize the deployment experience:

- **[Configure Docker Environment Variables](/docs/guides/docker-env-var-template/)** - Create customizable environment variable templates for Docker apps
- **[Configure Application Links](/docs/guides/application-links/)** - Create dynamic links to deployed applications using template variables
- **[Configuring Helm Charts for Distr](/docs/guides/helm-chart-registry-auth/)** - Set up authenticated image pulls with imagePullSecrets
- **[Manage Secrets](/docs/guides/secrets/)** - Store sensitive credentials centrally and reference them in deployments

## Automation & CI/CD

Automate your deployment pipeline with continuous integration and delivery:

- **[Automatic Deployments from GitHub](/docs/guides/automatic-deployments-from-github/)** - Automatically create versions and update all customer deployments on every release
- **[Pre-Flight Checks](/docs/guides/preflight-checks/)** - Validate target environments before deployment with custom shell scripts
- **[Kubernetes Compatibility Matrix](/docs/guides/kubernetes-compatibility-matrix/)** - Validate Helm charts against multiple Kubernetes versions and share compatibility reports with customers
- **[Vulnerability Scanning](/docs/guides/vulnerability-scanning/)** - Scan source dependencies and container images for known vulnerabilities and share reports with customers

## Customer Management

Manage customer access and their deployment experience:

- **[Customer Management](/docs/guides/customer-management/)** - Create customers, invite users, manage secrets, and control features in the Vendor Portal
- **[Application Entitlements](/docs/guides/application-entitlements/)** - Control which application versions customers can access and deploy
- **[Artifact Entitlements](/docs/guides/artifact-entitlements/)** - Control which artifacts and tags customers can access in your registry
- **[License Keys](/docs/guides/license-keys/)** - Issue signed JWT tokens your application can verify to enforce usage limits, seat counts, and feature access
- **[End-Customer View of Distr](/docs/guides/container-registry-for-end-customers/)** - Understanding the customer portal experience
- **[Support Bundles](/docs/product/support-bundles/)** - Let customers collect and share diagnostic data with your support team
